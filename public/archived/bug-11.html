Open Online Python IDE

Debugging Tips

  • Default start: range(5) starts at 0, not 1
  • Range parameters: You can specify both start and stop values
  • Addition compensation: Is adding 1 the best solution?
  • Cleaner approach: Fix the range instead of the print statement

What You Should Learn

  • How range() works with one parameter vs two parameters
  • Default start value is 0 when not specified
  • Better to fix the source than compensate in output
  • Understanding range(start, stop) syntax
