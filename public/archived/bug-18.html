Buggy Code

shopping_list = ["milk", "bread", "eggs"]
empty_list = []

print("Checking if shopping list has items...")
if len(shopping_list) == 0:  # Bug: checking if empty, not non-empty
    print("Shopping list has items!")
    for item in shopping_list:
        print(f"- {item}")
else:
    print("Shopping list is empty.")

print("\nChecking empty list...")
if len(empty_list) == 0:  # This condition is correct for empty
    print("Empty list is indeed empty.")
else:
    print("Empty list has items.")
Open Online Python IDE

Debugging Tips

  • Logic error: Checking for empty (== 0) when you want non-empty (!= 0)
  • Alternative methods: Use 'if shopping_list:' for non-empty or 'if not shopping_list:' for empty
  • Length check: len(list) > 0 means non-empty, len(list) == 0 means empty
  • Pythonic way: Lists evaluate to True if non-empty, False if empty

What You Should Learn

  • Different ways to check if a list is empty or non-empty
  • The truthiness of lists in Python (empty lists are falsy)
  • Using len() function vs direct boolean evaluation
  • Understanding logical conditions and their opposites
