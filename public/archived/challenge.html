Challenge Goals

Each numbered bug has a specific goal. Fix the code to achieve these outcomes:

Bug 1:
The list 'tasks' must contain 0, 1, 2, 3. The buggy code will append the final loop value multiple times. HINT: Closure Scope
Bug 2:
Calling 'add_item' twice should result in two independent lists: [10], [10, 20]. The buggy code will merge them. HINT: Mutable Default Arguments
Bug 3:
The sum should precisely equal 0.3. The buggy code will exhibit floating point inaccuracies.
Bug 4:
Print the final dictionary which must contain only keys 1, 3, 5, 7, 9. The buggy code will skip elements. HINT: Modifying dict while iterating
Bug 5:
All numbers in the 'squares' list must be 4, 9, 16. The buggy code will incorrectly use the list comprehension variable.
Bug 6:
Print the keys 'a', 'b', 'c'. The buggy code prints the values.
Bug 7:
The 'count' must be 3. The buggy code stops the search early.
Bug 8:
The function 'process' should print the number and its square only if the number is even. The buggy code always prints both.
Bug 9:
The nested loop should print 9 pairs. The buggy code only prints 6.
Bug 10:
The list 'data' must contain only the first number [10]. The buggy code fails to copy the list correctly.
Bug 11:
Catch the IndexError exception when running the loop. The buggy code catches the wrong exception type.
Bug 12:
Only print the elements that are explicitly listed in the 'valid_numbers' tuple (3, 7). The buggy code uses 'in' incorrectly for the check.
Bug 13:
Change the first letter of 'word' to 'J' resulting in "Jello". The buggy code will cause a TypeError. HINT: Immutability
Bug 14:
The loop must run until the user enters a number greater than 10. The buggy code uses incorrect logic negation.
Bug 15:
Check if the number is between 10 and 20 (inclusive). The buggy code uses improper chained comparison.
Bug 16:
Find the first person whose age is over 30 and print their name. The buggy code prints the name of the last person.
Bug 17:
The code must print "Welcome, user!". The buggy code outputs "Access Denied" due to type mismatch.
Bug 18:
The 'else' block of the loop must run only if the loop completes without a break. The buggy code prints "Found" and still runs the else block.
Bug 19:
The final printed list must be [1, 2, 3, 4] with the modification reflected. The buggy code fails to use a non-shallow copy.
Bug 20:
The code must print the last element of the list, which is 'd'. The buggy code accesses the element incorrectly.

# Bug 1 (Closure Scope)
print("\n--- BUG 1: Closure Scope Trap ---")
tasks = []

    tasks.append(lambda: print("Buggy Task:", i))

for task in tasks:
    task()

# Bug 2 (Mutable Default Arguments)
print("\n--- BUG 2: Mutable Default Argument Trap ---")
def add_item(item, item_list=[]):
    item_list.append(item)
    return item_list

list_a = add_item(10)
list_b = add_item(20)
print("Buggy List A:", list_a)
print("Buggy List B:", list_b)

# Bug 4 (Modifying Dict while Iterating)
print("\n--- BUG 4: Modifying Dict While Iterating ---")
numbers = {1: 'one', 2: 'two', 3: 'three', 4: 'four', 5: 'five'}
try:
    for key in numbers.keys():
        if key % 2 == 0:
            del numbers[key]
    print("Buggy Final Dict:", numbers)
except RuntimeError as e:
    print("Buggy Code Caught Error:", e)

        break

    square_and_print()
