Question 2 Easy

What's the difference between these two functions?

def function_a(x): return x * 2 def function_b(x): print(x * 2) result1 = function_a(5) result2 = function_b(5) print(f"Result1: {result1}, Result2: {result2}")
A
Both functions work the same way
B
function_a returns 10, function_b prints 10 but returns None
C
function_a prints 10, function_b returns 10
D
Both functions return 10

üí° Explanation:

return sends a value back to the caller, while print() displays output but the function returns None. So result1 = 10, result2 = None, and function_b also prints 10 to console.

üí° Explanation:

Function annotations are hints about types but don't enforce them. They're stored in the __annotations__ attribute. The function still works with any types, calculating 100.0 * 0.1 = 10.0.

B
Doubled: [2, 4, 6, 8], Squared: [1, 4, 9, 16]
C
Doubled: [2, 4, 6, 8], Squared: [2, 4, 6, 8]
D
Error - cannot pass function as argument

üí° Explanation:

Higher-order functions can take other functions as parameters. apply_operation applies the given function to each element: double gives [2, 4, 6, 8], square gives [1, 4, 9, 16].
