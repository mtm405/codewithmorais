1. üöÄ Function Basics - The def Keyword

What is a Function?

A function is a reusable block of code that performs a specific task. Think of it like a recipe - you write it once, then use it many times!

# Basic function definition def say_hello(): print("Hello, World!") # Calling (using) the function say_hello() say_hello() # Can use it multiple times!
Hello, World! Hello, World!

2. üìã Function Parameters and Call Signatures

Function Call Signatures

The call signature defines what information a function needs (parameters) and what it gives back (return type).

# Function with parameters def greet_person(name, age): print(f"Hello {name}! You are {age} years old.") # Calling with arguments greet_person("Sarah", 16) greet_person("Mike", 17)
Hello Sarah! You are 16 years old. Hello Mike! You are 17 years old.

Call Signature: greet_person(name: str, age: int) -> None

Multiple Parameters Example

# Function with multiple parameters def calculate_grade(points_earned, total_points, student_name): percentage = (points_earned / total_points) * 100 print(f"{student_name} scored {percentage:.1f}%") # Different ways to call the function calculate_grade(85, 100, "Emma") calculate_grade(92, 100, "Josh")
Emma scored 85.0% Josh scored 92.0%

3. ‚Ü©Ô∏è Return Statements - Getting Values Back

Functions That Give Back Values

The return keyword lets functions send a value back to whoever called them. This makes functions much more powerful!

# Function that returns a value def add_numbers(a, b): sum_result = a + b return sum_result # Using the returned value answer = add_numbers(15, 25) print(f"The sum is: {answer}") # Can use directly in expressions total = add_numbers(10, 5) + add_numbers(3, 7) print(f"Grand total: {total}")
The sum is: 40 Grand total: 25

Return vs Print - Important Difference!

# Function that PRINTS (doesn't return) def print_double(x): print(x * 2) # Function that RETURNS def get_double(x): return x * 2 # Compare the difference result1 = print_double(5) # Prints 10, but result1 is None result2 = get_double(5) # Returns 10, result2 is 10 print(f"result1 is: {result1}") print(f"result2 is: {result2}")
10 result1 is: None result2 is: 10

4. üéõÔ∏è Default Parameter Values

Making Parameters Optional

Default values let you make some parameters optional. If the caller doesn't provide a value, Python uses the default!

# Function with default parameters def create_profile(name, age=16, grade="10th"): return f"Name: {name}, Age: {age}, Grade: {grade}" # Different ways to call it print(create_profile("Alex")) # Uses defaults print(create_profile("Sarah", 17)) # Custom age, default grade print(create_profile("Mike", 15, "9th")) # All custom values
Name: Alex, Age: 16, Grade: 10th Name: Sarah, Age: 17, Grade: 10th Name: Mike, Age: 15, Grade: 9th

Practical Example: Text Formatter

def format_message(text, prefix="INFO", uppercase=False): if uppercase: text = text.upper() return f"[{prefix}] {text}" # Various ways to use it print(format_message("System started")) print(format_message("Error occurred", "ERROR")) print(format_message("Warning message", "WARN", True))
[INFO] System started [ERROR] Error occurred [WARN] WARNING MESSAGE

5. ‚è∏Ô∏è The Pass Statement - Placeholders

Function Stubs with Pass

The pass statement is a placeholder - it does nothing but allows you to create function "stubs" that you'll fill in later.

# Function stubs for a game we're planning def start_game(): pass # Will implement later def get_player_move(): pass # TODO: Get input from player def check_winner(board): pass # TODO: Check game state # These functions exist but don't do anything yet print("Functions defined successfully!")
Functions defined successfully!

Gradually Building Functions

# Start with a stub def analyze_text(text): pass # Add some basic functionality def analyze_text(text): word_count = len(text.split()) return word_count # Keep expanding it def analyze_text(text): word_count = len(text.split()) char_count = len(text) avg_word_length = char_count / word_count if word_count > 0 else 0 return { "words": word_count, "characters": char_count, "avg_word_length": avg_word_length }
