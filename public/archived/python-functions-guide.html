Lesson 1: The Basics of def and Arguments üõ†Ô∏è

Why Use Functions?

Functions are like mini-programs that perform specific tasks. They help keep code organized (modularity) and allow us to reuse the same code block without rewriting it.

Key Syntax: Defining a Function

The def keyword is used to define a function, followed by a name, parentheses for arguments, and a colon.

Lesson 2: The Power of return üîÑ

print vs. return

üñ®Ô∏è print()

What it does: Shows a result to the user in the console. The value is not stored for later use.

Returns: None

‚Ü©Ô∏è return

What it does: Sends a value back to the place where the function was called. This value can be stored in a variable or used in other calculations.

Returns: The specified value

Key Syntax: Returning a Value

Use the return keyword to specify the output of the function.

# Function that returns a value def add_numbers(a, b): result = a + b return result # Using the returned value sum_result = add_numbers(5, 3) print(f"The sum is: {sum_result}")
The sum is: 8

Lesson 3: Scope and Best Practices üñºÔ∏è

Local vs. Global Scope

Local Scope: Variables defined inside a function (including its arguments) are local. They exist only while the function is running and cannot be accessed outside it.

Global Scope: Variables defined outside any function are global and can be read anywhere in the code.

Best Practice: Avoid using global variables inside functions. Always pass data into a function using arguments and get data out using return.

Task: Predict and explain the outcome of the following code.

Fix: Rewrite the change_value function so that it correctly updates the my_number in the global scope using a return statement.

Initial Code (Illustrates Local Scope):

my_number = 10 # This is a GLOBAL variable def change_value(num): # 'new_num' is a LOCAL variable, defined only here new_num = num + 5 print(f"Inside function, new_num is: {new_num}") change_value(my_number) # The line below is the key test: print(f"Outside function, my_number is still: {my_number}") # Why didn't my_number change? (Answer: Local scope) # Try to print(new_num) here and see what happens (Answer: NameError)
Inside function, new_num is: 15 Outside function, my_number is still: 10
Why didn't my_number change? Because new_num is a local variable that only exists inside the function. The global my_number remains unchanged.

Fixed Code (Using return):

my_number = 10 def change_value(num): # Calculate the new value new_num = num + 5 # Return the new value return new_num # Call the function and assign the RETURNED value back to the global variable my_number = change_value(my_number) print(f"Outside function, my_number is now: {my_number}")
Outside function, my_number is now: 15
Key Learning: To modify a global variable, return the new value from the function and assign it back to the global variable.

    # Check what unit we're converting FROM
    if unit == "C":
        # Convert from Celsius to Fahrenheit

‚ö° The Magic any() Function

The any() function is like having a super-smart assistant that checks multiple things at once!

What it does: Returns True if ANY item in a list/sequence is True 
# Example: Check if password has ANY uppercase letters

# Long way (what any() does internally):
has_upper = False
for char in password:
    if char.isupper():
        has_upper = True
        break

# Short way using any():
has_upper = any(char.isupper() for char in password)

üèóÔ∏è Password Validation Template
def validate_password(password):
    # Check length first (easiest check)
    if len(password) < 8:
        return "Weak - Too short"
    
    # Check for uppercase letters
    if not any(char.isupper() for char in password):
        return "Weak - Missing uppercase"
    
    # Check for lowercase letters  
    if not any(char.islower() for char in password):
        return "Weak - Missing lowercase"
    
    # Check for digits
    if not any(char.isdigit() for char in password):
        return "Weak - Missing digit"
    
    # If all checks pass
    return "Strong"

üéØ What is Function Composition?

Function composition means calling one function from inside another function. It's like having a team where each person has a specific job, and they work together to accomplish a bigger task!

üë§
Helper Functions
Do one specific task very well
üë•
Main Function
Coordinates all the helpers
üèÜ
Final Result
Complex task completed!
üîç Step-by-Step Example

Let's build our renovation calculator step by step:

# Step 1: Create a helper function
def calculate_area(length, width):
    """Calculate rectangle area - this function has ONE job"""
    area = length * width
    return area

# Step 2: Create the main function that USES the helper
def room_renovation_cost(length, width, cost_per_sqft):
    """Calculate total cost - this function coordinates everything"""
    
    # Call the helper function and store its result
    room_area = calculate_area(length, width)
    
    # Use the helper's result in our calculation
    total_cost = room_area * cost_per_sqft
    
    # Return our final answer
    return total_cost

# Step 3: Test both functions
area_result = calculate_area(10, 12)        # Helper works independently
total_result = room_renovation_cost(10, 12, 15)  # Main function uses helper

print(f"Area: {area_result} sq ft")         # 120
print(f"Total cost: ${total_result}")       # $1800
