üêõ Most Common Python Errors

1. SyntaxError

What it means: Python can't understand your code structure

# Missing colon if x > 5 print("Greater than 5") # Unmatched parentheses print("Hello World" # Wrong indentation def my_function(): print("This is wrong")

üí° How to Fix:

  • Check for missing colons after if, while, for, def statements
  • Ensure all parentheses, brackets, and quotes are matched
  • Use consistent indentation (4 spaces recommended)
  • Check for reserved keywords used as variable names
Quick Fix: Most IDEs highlight syntax errors in real-time. Look for red underlines!

2. IndentationError

What it means: Python can't determine your code blocks

def my_function(): if True: x = 1 y = 2 # Wrong indentation level # Mixed tabs and spaces def another_function(): if True: x = 1 # 4 spaces y = 2 # Tab character - ERROR!

üí° How to Fix:

  • Use only spaces OR only tabs (spaces recommended)
  • Use 4 spaces per indentation level
  • Make sure all code in the same block has the same indentation
  • Configure your editor to show whitespace characters

3. NameError

What it means: You're using a variable that doesn't exist

print(my_variable) # NameError: name 'my_variable' is not defined def my_function(): print(x) # x is not defined in this scope # Typo in variable name name = "Alice" print(nam) # NameError: name 'nam' is not defined

üí° How to Fix:

  • Make sure variables are defined before use
  • Check for typos in variable names
  • Understand variable scope (local vs global)
  • Import modules before using their functions

4. TypeError

What it means: You're using the wrong data type for an operation

# Can't add string and number result = "5" + 3 # TypeError # Can't use string as function name = "Alice" name() # TypeError: 'str' object is not callable # Wrong argument type len(123) # TypeError: object of type 'int' has no len()

üí° How to Fix:

  • Convert between types: int(), str(), float()
  • Check data types with type() or isinstance()
  • Use appropriate operations for each type
  • Read error messages carefully - they tell you what's wrong

5. IndexError

What it means: You're trying to access a list/string position that doesn't exist

my_list = [1, 2, 3] print(my_list[5]) # IndexError: list index out of range # Common in loops for i in range(10): print(my_list[i]) # Error when i >= 3

üí° How to Fix:

  • Check list length with len() before accessing
  • Use range(len(my_list)) in loops
  • Handle edge cases (empty lists)
  • Use try-except for uncertain scenarios

6. KeyError

What it means: You're trying to access a dictionary key that doesn't exist

person = {"name": "Alice", "age": 30} print(person["height"]) # KeyError: 'height' # Common when processing data for item in data: print(item["required_field"]) # Error if key missing

üí° How to Fix:

  • Use dict.get() with default values
  • Check if key exists with 'key' in dict
  • Use try-except KeyError blocks
  • Validate data structure before processing

1. Print Statement Debugging

The most basic but effective debugging method

def calculate_average(numbers): print(f"DEBUG: Input numbers: {numbers}") print(f"DEBUG: Type of numbers: {type(numbers)}") total = sum(numbers) print(f"DEBUG: Total sum: {total}") count = len(numbers) print(f"DEBUG: Count: {count}") if count == 0: print("DEBUG: Division by zero prevented!") return 0 average = total / count print(f"DEBUG: Final average: {average}") return average

2. Python Debugger (pdb)

Interactive debugging with breakpoints and step execution

import pdb def problematic_function(data): pdb.set_trace() # Execution will pause here for item in data: processed = process_item(item) results.append(processed) return results # pdb commands: # n (next line) # s (step into functions) # c (continue execution) # l (list current code) # p variable_name (print variable) # q (quit debugger)

3. Assert Statements

Check assumptions and catch errors early

def divide_numbers(a, b): assert isinstance(a, (int, float)), f"a must be a number, got {type(a)}" assert isinstance(b, (int, float)), f"b must be a number, got {type(b)}" assert b != 0, "Cannot divide by zero" result = a / b assert result is not None, "Result should not be None" return result

4. Logging for Better Debugging

Professional debugging with different log levels

import logging # Setup logging logging.basicConfig(level=logging.DEBUG, format='%(asctime)s - %(levelname)s - %(message)s') def process_data(data): logging.info(f"Starting to process {len(data)} items") try: for i, item in enumerate(data): logging.debug(f"Processing item {i}: {item}") result = complex_operation(item) logging.debug(f"Result for item {i}: {result}") except Exception as e: logging.error(f"Error processing item {i}: {e}") logging.exception("Full traceback:") raise logging.info("Processing completed successfully")

5. Exception Handling for Debugging

Catch and analyze errors without crashing

import traceback def debug_wrapper(func): """Decorator to add debugging to any function""" def wrapper(*args, **kwargs): try: print(f"Calling {func.__name__} with args={args}, kwargs={kwargs}") result = func(*args, **kwargs) print(f"{func.__name__} returned: {result}") return result except Exception as e: print(f"ERROR in {func.__name__}: {type(e).__name__}: {e}") print("Full traceback:") traceback.print_exc() raise return wrapper @debug_wrapper def risky_function(x, y): return x / y # Might fail with division by zero

üìö Complete Error Reference

AttributeError

Cause: Accessing attribute/method that doesn't exist

text = "hello" text.append("world") # strings don't have append method
Fix: Check available methods with dir(object) or help(object)

ImportError / ModuleNotFoundError

Cause: Module cannot be imported

import non_existent_module # ModuleNotFoundError from math import non_existent_function # ImportError
Fix: Install module with pip, check spelling, verify module path

ValueError

Cause: Correct type but inappropriate value

int("hello") # Can't convert "hello" to integer math.sqrt(-1) # Can't take square root of negative number
Fix: Validate input values before conversion/calculation

ZeroDivisionError

Cause: Division by zero

result = 10 / 0 # Direct division by zero result = x / (y - y) # Indirect division by zero
Fix: Check divisor before division, use try-except

FileNotFoundError

Cause: Trying to open non-existent file

with open("missing_file.txt", "r") as f: content = f.read()
Fix: Check if file exists with os.path.exists(), use try-except

RecursionError

Cause: Maximum recursion depth exceeded

def infinite_recursion(n): return infinite_recursion(n + 1) # No base case!
Fix: Add proper base case, consider iterative solution

UnicodeError

Cause: Text encoding/decoding issues

text = "caf√©" encoded = text.encode('ascii') # UnicodeEncodeError
Fix: Use UTF-8 encoding, handle encoding errors explicitly

MemoryError

Cause: Not enough memory for operation

huge_list = [0] * (10 ** 10) # Too much memory needed
Fix: Process data in chunks, use generators, optimize algorithms
