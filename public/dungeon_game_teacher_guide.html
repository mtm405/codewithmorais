üé≤ Role 1: Dice Master (dice_master.py)

Purpose: Provide random numbers for combat rolls

Imports Needed: import random

Functions to Write: 1 function that simulates a d20 roll

Complete Solution

import random

def roll_d20():
    """
    Simulates rolling a 20-sided die.
    Returns a number between 1 and 20.
    """
    return random.randint(1, 20)

def calculate_damage(strength):
    """
    Calculates weapon damage based on enemy strength.
    A weapon does 5-10 base damage plus enemy strength bonus.
    Returns the total damage.
    """
    base_damage = random.randint(5, 10)
    total_damage = base_damage + strength
    return total_damage

Key Learning Points

  • random.randint(a, b) returns a random integer from a to b (inclusive)
  • Functions RETURN values; they don't print them
  • Each function does ONE job only
  • Docstrings (the text in triple quotes) explain what the function does

What Students Should See When Testing

>>> from dice_master import roll_d20, calculate_damage
>>> roll_d20()
17
>>> roll_d20()
5
>>> calculate_damage(3)
12
>>> calculate_damage(3)
8

‚ùå Common Mistakes

  • Printing instead of returning: print(random.randint(1, 20)) - This should be return random.randint(1, 20)
  • Wrong range: Using (1, 21) instead of (1, 20) - d20 means 20 sides!
  • Forgetting import: Using random.randint() without import random first

üí° Teaching Tips

Have students test their functions in Python shell before moving on. They should see different numbers each time they call the function. If they see the same number twice, they need to understand why that's OK (random chance)!

üëπ Role 2: Encounter Designer (encounters.py)

Purpose: Create enemy data and select random encounters

Imports Needed: import random

Functions to Write: 1-2 functions that manage enemy encounters

Complete Solution

import random

def get_all_encounters():
    """
    Returns a list of all possible enemies in the dungeon.
    Each enemy is a dictionary with name, hp, and strength.
    """
    encounters = [
        {"name": "Goblin", "hp": 15, "strength": 2},
        {"name": "Orc", "hp": 20, "strength": 3},
        {"name": "Dark Knight", "hp": 25, "strength": 4},
        {"name": "Skeleton", "hp": 12, "strength": 2},
        {"name": "Troll", "hp": 30, "strength": 5}
    ]
    return encounters

def get_random_encounter():
    """
    Selects a random enemy from all encounters.
    Returns one enemy dictionary.
    """
    all_enemies = get_all_encounters()
    random_enemy = random.choice(all_enemies)
    # Make a copy so multiple calls don't modify the same object
    return random_enemy.copy()

Key Learning Points

  • Dictionaries store multiple related pieces of data together
  • Lists hold multiple dictionaries
  • random.choice() picks one random item from a list
  • .copy() creates a duplicate so changing one doesn't affect the original
  • Functions can call other functions

What Students Should See When Testing

>>> from encounters import get_random_encounter
>>> enemy = get_random_encounter()
>>> enemy["name"]
"Orc"
>>> enemy["hp"]
20
>>> get_random_encounter()["name"]
"Goblin"

‚ùå Common Mistakes

  • Wrong dictionary syntax: Using enemy.name instead of enemy["name"]
  • Not copying the enemy: Returning the original dictionary means all games modify the same object
  • Forgetting square brackets: random.choice needs a list, and lists use [ ]

üí° Teaching Tips

Have students understand WHY we need .copy(). Have them test without it: they'll see the enemy HP stay changed across multiple games! This teaches why we need to be careful with shared data.

üí∞ Role 3: Treasure Keeper (treasure.py)

Purpose: Calculate rewards when enemies are defeated

Imports Needed: import random

Functions to Write: 1-2 functions that calculate rewards

Complete Solution

import random

def calculate_loot(enemy_hp, enemy_strength):
    """
    Calculates how much gold a defeated enemy drops.
    Stronger enemies drop more gold.
    """
    base_gold = 50
    bonus_gold = enemy_hp + (enemy_strength * 10)
    total_gold = base_gold + bonus_gold
    return total_gold

def calculate_xp(enemy_hp, enemy_strength):
    """
    Calculates experience points gained from defeating an enemy.
    XP is based on the enemy's difficulty.
    """
    base_xp = 25
    xp_multiplier = enemy_strength
    total_xp = base_xp * xp_multiplier
    return total_xp

def get_treasure(enemy_name, enemy_hp, enemy_strength):
    """
    Main function that returns both loot and XP.
    Returns a dictionary with all rewards.
    """
    gold = calculate_loot(enemy_hp, enemy_strength)
    xp = calculate_xp(enemy_hp, enemy_strength)
    
    treasure = {
        "enemy": enemy_name,
        "gold": gold,
        "xp": xp,
        "message": f"Defeated {enemy_name}! Earned {gold} gold and {xp} XP!"
    }
    return treasure

Key Learning Points

  • Helper functions (like calculate_loot) do smaller jobs
  • Main functions (like get_treasure) use helper functions
  • Math can be part of game logic
  • f-strings (f"text {variable}") embed variables in text
  • Functions can return multiple values together in a dictionary

What Students Should See When Testing

>>> from treasure import get_treasure
>>> reward = get_treasure("Orc", 20, 3)
>>> reward["gold"]
115
>>> reward["xp"]
75
>>> reward["message"]
"Defeated Orc! Earned 115 gold and 75 XP!"

‚ùå Common Mistakes

  • Forgetting parameters: Functions need all the data they need passed in
  • Wrong math order: Make sure multiplication happens before addition (it does automatically in Python!)
  • Not returning the dictionary: Building treasure but not returning it

üí° Teaching Tips

Challenge students: "What if we wanted Trolls to give 2x loot?" They need to see they'd have to add a parameter or look up the enemy type. This teaches why game designers care about data structure!

üéÆ Role 4: Game Master (main.py)

Purpose: Import all modules and run the game

Imports Needed: All three modules + random module

What to Write: Game loop, combat logic, and player interaction

Complete Solution

import random
from dice_master import roll_d20, calculate_damage
from encounters import get_random_encounter
from treasure import get_treasure

def run_game():
    """Main game loop. This is where it all comes together!"""
    
    print("=" * 50)
    print("‚öîÔ∏è WELCOME TO THE ENDLESS DUNGEON ‚öîÔ∏è")
    print("=" * 50)
    print()
    
    # Game state
    player_hp = 50
    game_over = False
    
    # Get a random enemy
    enemy = get_random_encounter()
    print(f"A wild {enemy['name']} appears!")
    print(f"Enemy HP: {enemy['hp']} | Enemy Strength: {enemy['strength']}")
    print()
    
    # Combat loop
    while not game_over and player_hp > 0 and enemy['hp'] > 0:
        print("-" * 50)
        print(f"Your HP: {player_hp} | Enemy HP: {enemy['hp']}")
        print("-" * 50)
        
        # Player's turn
        print("Your turn! Rolling to attack...")
        player_roll = roll_d20()
        print(f"üé≤ You rolled: {player_roll}")
        
        if player_roll >= 10:
            damage = calculate_damage(0)  # Player base damage
            enemy['hp'] -= damage
            print(f"‚úÖ HIT! You dealt {damage} damage!")
        else:
            print("‚ùå MISS! Your attack went wide!")
        
        # Check if enemy is defeated
        if enemy['hp'] <= 0:
            print()
            print("üéâ VICTORY!")
            reward = get_treasure(enemy['name'], enemy['hp'] + damage, enemy['strength'])
            print(reward['message'])
            game_over = True
            break
        
        print()
        
        # Enemy's turn
        print(f"{enemy['name']}'s turn...")
        enemy_roll = roll_d20()
        print(f"üé≤ {enemy['name']} rolled: {enemy_roll}")
        
        if enemy_roll >= 10:
            damage = calculate_damage(enemy['strength'])
            player_hp -= damage
            print(f"üí• HIT! You took {damage} damage!")
        else:
            print("üõ°Ô∏è MISSED! You dodged the attack!")
        
        # Check if player is defeated
        if player_hp <= 0:
            print()
            print("üíÄ DEFEAT!")
            print(f"The {enemy['name']} defeated you...")
            game_over = True
        
        print()
    
    print("=" * 50)
    print("Thanks for playing!")
    print("=" * 50)

# Run the game
if __name__ == "__main__":
    run_game()

Key Learning Points

  • Imports bring in code from other files
  • The game master USES functions from other modules, doesn't rewrite them
  • While loops let code run multiple times until a condition is false
  • Modifying dictionaries: enemy['hp'] -= damage changes the stored value
  • The if __name__ == "__main__": line is how Python knows to run the game

‚ùå Common Mistakes

üêõ Debugging Guide for Students

Error: "ModuleNotFoundError: No module named 'dice_master'"

Cause: Python can't find the file

Solution: Make sure all four .py files are in the SAME folder. Check the spelling is exact!

Error: "NameError: name 'roll_d20' is not defined"

Cause: Forgot to import the function

Solution: Add from dice_master import roll_d20 at the top of main.py

Game plays but keeps rolling forever

Cause: While loop condition never becomes false (game_over never set to True)

Solution: Make sure you set game_over = True when enemy HP or player HP reaches 0

Values aren't changing in dictionaries

Cause: Modifying a copy instead of the original

Solution: Make sure you're using enemy['hp'] -= damage (modifies the dict) not just damage = damage + 5 (creates a new variable)
