A. Mastering Conditional Logic (if/elif/else)

The primary risk here is mismanaging the execution order and failing to define variables in all paths.

Key Concept: Mutually Exclusive Paths

In Python, the if/elif/else structure guarantees that only one block of code will execute. This is why order and operators are critical.

Condition Operator Example Use Case (Q38: Grading)
Specific Value == if grade == 100: (Only 100 is Outstanding)
Upper Range >= elif grade >= 90: (Handles 90-99, since 100 was already handled)
Lower Range <= elif grade <= 70: (Handles 70 and below)
Catch-all else: Catches everything not met above (71-89)

üí° Key Takeaway: Always ensure that variables are defined in ALL possible code paths before they are used!

B. Deep Dive: Loop Structures (for and while)

Loops are the foundation of repeated actions. The key is knowing which one to use and how to control its iteration.

for Loops and range()

The for loop is ideal when you know the number of iterations or are looping through a collection (list, string, etc.).

Remember: range(start, stop) - The stop value is exclusive.
To iterate 5 times (weeks 1 to 5), you need range(1, 6).

break Statement

The break keyword immediately terminates the innermost loop.

Example (Q35):

cities = ['Anchorage', 'Juneau', 'Fairbanks', 'Ketchikan', 'Sitka', 'Wasilla'] for city in cities: print(f'{city} is a famous Alaskan city.') # This executes FIRST if city == 'Ketchikan': break # This stops the loop
Since the print() happens before the break is checked, 'Ketchikan' is printed, resulting in 4 cities being printed total.

while Loops for Validation (Q29)

A while loop runs indefinitely until its condition becomes False.

Goal: Keep asking for input until the response is valid.

location = ["North", "South", "West", "East"] # Must get the first response to initialize the variable response = input("Enter North, South, West, or East for a location: ") # The loop continues AS LONG AS the response is NOT in the list while response not in location: # The key is 'not in' print("Try again.") # Must ask for input again INSIDE the loop response = input("Enter North, South, West, or East for a location: ") print(response)
üí° Key Points:
  • Must initialize the variable BEFORE the while loop
  • Use not in to check if value is NOT in the list
  • Must ask for input again INSIDE the loop

üìÅ File Modes Quick Reference:
  • 'r' - Read only (default)
  • 'w' - Write (overwrites existing file)
  • 'a' - Append (adds to end of file)
  • 'x' - Create (fails if file exists)
