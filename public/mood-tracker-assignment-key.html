Section 1: Core I/O Structure and Modes (8 Points)

1. Context Manager (2 pts): What is the single most important reason for using the with open(...) structure instead of just calling f = open(...) and later f.close()?
Answer:
It guarantees the file is always closed, even if an error (exception) occurs while the program is reading or writing the file. This prevents file corruption and resource leaks.
2. File Modes (2 pts): Explain the crucial difference between the file modes 'w' (write) and 'a' (append).
Answer:
'w' (write) mode creates a new file or overwrites all existing content if the file already exists. 'a' (append) mode adds new content to the very end of the file, preserving everything already there.
3. Catastrophic Failure (2 pts): Why would using the 'w' mode every time a user logs a mood be a catastrophic failure for the application?
Answer:
Using 'w' would erase all previous log history every time the user ran the function, leaving only the single, newest log entry in the file.
4. Reading Methods (2 pts): The view_history function uses f.read(). If the log file was 100 lines long, how many separate strings would f.read() return?
Answer:
One (1). The f.read() method reads the entire file content as a single, large string.

Section 2: Defensive Programming and Formatting (8 Points)

5. Error Prevention (2 pts): What specific Python error is the line if not os.path.exists(FILENAME): in the view_history function designed to prevent if the log file is missing?
Answer:
It prevents the FileNotFoundError that would occur if the program tried to open a file in read mode ('r') that doesn't exist.
6. Logic Flow (2 pts): In the log_mood_entry function, what is the condition that determines whether the program uses 'w' mode (to create the header) or 'a' mode (to append the new entry)?
Answer:
The condition is if not os.path.exists(FILENAME):. If the file is missing (True), it uses 'w' to create the header. If the file is found (False), it proceeds to use 'a'.
7. Newline Character (2 pts): Why is the newline character (\n) added to the log_entry string? What would happen to the file contents if this character was forgotten in append mode?
Answer:
It ensures that each new entry starts on a fresh line in the text file. If forgotten, all subsequent entries would be written immediately after the previous line without a line break, making the file unreadable.
8. Robustness (2 pts): The main_menu function uses a try/except block. What is the primary benefit of using try/except here?
Answer:
It prevents the program from crashing (or raising a ValueError) if the user enters non-numeric text (like "exit" or "a") instead of the expected menu options (1, 2, or 3).
