üìù Detailed Information and Examples

A. Core Data Types (Objective 1.1)

Python is a dynamically-typed language, meaning variables automatically get their type based on the value assigned. You don't need to explicitly declare types like in Java or C++. Understanding data types is crucial because they determine what operations you can perform and how Python interprets your code.

The four fundamental data types you must master are:

  • ‚Ä¢ Integer (int): Whole numbers without decimals. Can be positive, negative, or zero. Example: x = 5, age = -10, count = 0
  • ‚Ä¢ Float (float): Numbers with decimal points, used for precise calculations. Example: y = 5.0, pi = 3.14159, temp = -2.5
  • ‚Ä¢ String (str): Sequence of characters enclosed in quotes (single, double, or triple). Used for text. Example: name = "Alice", msg = 'Hello'
  • ‚Ä¢ Boolean (bool): Represents truth values - only True or False. Used in conditions and logic. Example: is_valid = True, has_permission = False

üí° Why Data Types Matter:

  • ‚Ä¢ Different types support different operations ("2" + "3" = "23" vs 2 + 3 = 5)
  • ‚Ä¢ Type errors are common bugs ("age" - 5 will fail because you can't subtract from a string)
  • ‚Ä¢ The type() function helps you check what type a variable is
  • ‚Ä¢ You can convert between types using int(), float(), str(), and bool()
Example 1: Type Evaluation
a = 10 b = 10.5 c = "10" d = a == b # Evaluate types: print(type(a)) # <class 'int'> print(type(b)) # <class 'float'> print(type(c)) # <class 'str'> print(type(d)) # <class 'bool'>

B. Data and Data Type Operations (Objective 1.2)

Once you understand data types, you need to know how to manipulate and transform them. Python provides powerful built-in operations that work differently depending on the data type. Mastering these operations is essential for the certification exam.

Key Operations You Must Know:

  • ‚Ä¢ Type Conversion (Casting): Transform data from one type to another using int(), float(), str(), bool(). Common exam question: What happens when you convert "123" vs "12.5" to int?
  • ‚Ä¢ Indexing: Access individual elements by position. Remember: Python uses zero-based indexing (first item is index 0). Negative indices count from the end (-1 is last item).
  • ‚Ä¢ Slicing: Extract portions using [start:stop:step] notation. The stop index is NOT included. Example: "Python"[0:3] gives "Pyt"
  • ‚Ä¢ List Operations: Build and modify data structures with methods like append() (add to end), insert() (add at position), remove() (delete value), sort() (order items)
  • ‚Ä¢ Built-in Functions: Use len() (length), max() (largest value), min() (smallest value), sum() (total of numbers)

‚ö†Ô∏è Common Pitfalls:

  • ‚Ä¢ Converting "12.5" directly to int fails - convert to float first
  • ‚Ä¢ Slicing [2:5] gives indices 2, 3, 4 (NOT 5!)
  • ‚Ä¢ Strings are immutable - you can't change them directly, only create new ones
  • ‚Ä¢ remove() deletes the first occurrence only, not all occurrences
Example 2: Conversion and Slicing
# Data Type Conversion str_num = "42" int_num = int(str_num) print(int_num) # Converts '42' (str) to 42 (int) # Indexing and Slicing (Applies to strings and lists) my_list = ['A', 'B', 'C', 'D', 'E'] print(my_list[0]) # Indexing: 'A' print(my_list[1:4]) # Slicing: ['B', 'C', 'D'] # List Operations (Construct data structures) my_list.append('F') my_list.insert(1, 'Z') my_list.remove('A') my_list.sort() print(max(my_list)) print(my_list)

C. Operator Precedence and Selection (Objectives 1.3 & 1.4)

Operators are grouped by precedence (high to low):

  • 1. Arithmetic: ** / * // % + -
  • 2. Comparison: == >= <= != > <
  • 3. Logical: not, and, or
  • 4. Identity/Containment: is, in
Example 3: Precedence and Operator Selection
# Operator Precedence # 1. ** (Exponentiation) -> 2. * / // % -> 3. + - result = 10 + 4 * 2 ** 3 print(result) # 42 (4 * 8 = 32, then 10 + 32 = 42) # Operator Selection (Comparison and Logical) score = 85 is_passing = score >= 70 and score <= 100 print(is_passing) # True # Identity (is) and Containment (in) list1 = [1, 2, 3] list2 = [1, 2, 3] print(list1 is list2) # False (different objects in memory) print(3 in list1) # True (3 is in the list)
