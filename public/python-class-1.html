üéØ What Are Data Types?

In Python, every value has a type that determines:

  • ‚úÖ What operations you can perform
  • ‚úÖ How Python stores the value in memory
  • ‚úÖ What functions work with that value
üí° Example: The value 5 is an integer, so you can add it to another number. But the text "5" is a string, so you can't do math with it directly!

‚úÖ This Works

5 + 3 = 8

Adding two numbers

‚ùå This Fails

"5" - 3 = Error!

Can't subtract from text

üî¢ The Four Core Data Types

1. Integer (int) - Whole Numbers

Numbers without decimal points. Can be positive, negative, or zero.

age = 17 count = -5 zero = 0 big_num = 1000000

2. Float (float) - Decimal Numbers

Numbers with decimal points. Used when precision matters.

price = 19.99 temperature = -2.5 pi = 3.14159

3. String (str) - Text

Any text enclosed in quotes. Use single, double, or triple quotes.

name = "Alice" greeting = 'Hello World' message = """This is a multi-line string"""

4. Boolean (bool) - True or False

Only two possible values. Used for decisions and logic.

is_student = True is_complete = False passed_exam = True

üîç How to Identify Data Types

Use the type() function:

value = 42 print(type(value)) # <class 'int'> value = 42.5 print(type(value)) # <class 'float'> value = "42" print(type(value)) # <class 'str'> value = True print(type(value)) # <class 'bool'>
‚ö†Ô∏è Key Difference: 42 (integer) vs "42" (string)

Even though they look similar, they're completely different types! The quotes make all the difference.

üîÑ Type Conversion (Casting)

You can convert values from one type to another using these functions:

Common Conversions:
# String to Integer num = int("42") # "42" becomes 42 # Integer to String text = str(99) # 99 becomes "99" # String to Float decimal = float("3.14") # "3.14" becomes 3.14 # Anything to Boolean result = bool(1) # 1 becomes True result = bool(0) # 0 becomes False result = bool("") # Empty string becomes False
‚ö†Ô∏è Common Mistake: Converting "12.5" directly to int fails!
# This causes an ERROR: num = int("12.5") # ‚ùå Can't convert string with decimal directly # Correct way: num = int(float("12.5")) # ‚úÖ Convert to float first, then to int # Result: 12

‚ûï Arithmetic Operators

Use these operators to perform mathematical calculations:

+ (Addition)

5 + 3 # Result: 8 "Hello" + " World" # Result: "Hello World"

- (Subtraction)

10 - 3 # Result: 7 -5 - 2 # Result: -7

* (Multiplication)

4 * 5 # Result: 20 "Ha" * 3 # Result: "HaHaHa"

/ (Division)

10 / 2 # Result: 5.0 (always float!) 15 / 4 # Result: 3.75

** (Power)

2 ** 3 # Result: 8 (2 to the power of 3) 5 ** 2 # Result: 25

// (Floor Division)

15 // 4 # Result: 3 (rounds down) 10 // 3 # Result: 3
üí° Remember: Division (/) always gives a float, even if the result is a whole number. Use floor division (//) to get an integer.

‚öñÔ∏è Operator Precedence (Order of Operations)

PEMDAS in Python: Parentheses, Exponents, Multiplication/Division, Addition/Subtraction

Order (from highest to lowest priority):
  1. ** (Exponent)
  2. *, /, //, % (Multiply, Divide, Floor Divide, Modulo)
  3. +, - (Add, Subtract)
Example 1:
result = 10 + 4 * 2 ** 3 # Step by step: # 1. 2 ** 3 = 8 (exponent first) # 2. 4 * 8 = 32 (multiplication next) # 3. 10 + 32 = 42 (addition last) print(result) # Output: 42
Example 2 (With Parentheses):
result = (10 + 4) * 2 ** 3 # Step by step: # 1. (10 + 4) = 14 (parentheses first!) # 2. 2 ** 3 = 8 (exponent next) # 3. 14 * 8 = 112 (multiplication last) print(result) # Output: 112
üí° Use Parentheses: When in doubt, use parentheses to make the order clear! It helps prevent errors and makes your code easier to read.

üîÄ Comparison Operators

Compare values and get True or False as a result:

== (Equal)

5 == 5 # True 5 == 3 # False "hi" == "hi" # True

!= (Not Equal)

5 != 3 # True 5 != 5 # False "a" != "b" # True

> (Greater Than)

10 > 5 # True 3 > 5 # False 10 > 10 # False

< (Less Than)

3 < 5 # True 10 < 5 # False 5 < 5 # False

>= (Greater or Equal)

10 >= 10 # True 5 >= 10 # False 15 >= 10 # True

<= (Less or Equal)

5 <= 10 # True 5 <= 5 # True 10 <= 5 # False

‚öôÔ∏è Logical Operators

Combine multiple conditions using and, or, and not:

and - Both conditions must be True

age = 18 has_license = True result = age >= 18 and has_license # True (both are true) score = 65 result = score > 70 and score < 100 # False (first is false)

or - At least one condition must be True

day = "Saturday" is_weekend = day == "Saturday" or day == "Sunday" # True result = 10 > 20 or 5 < 10 # True (second is true)

not - Reverses the result

is_rainy = True is_sunny = not is_rainy # False result = not (5 > 10) # True (5 > 10 is False, then not False = True)

üåç Real-World Example

Movie Theater Age Check

You want to watch an R-rated movie. Theater requires you to be 17+ or have a parent.

age = 16 has_parent = True # Check if eligible to watch can_watch = age >= 17 or has_parent # True (has parent) if can_watch: print("Welcome! Enjoy the movie!") # This will print else: print("You can't watch this movie")
‚úÖ Why Logical Operators Matter:

Real programs need to make decisions based on multiple conditions. Logical operators let us combine conditions to create realistic rules.
