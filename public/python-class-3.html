üìù Detailed Information and Examples

A. File Input and Output (Objective 3.1)

File operations involve open, close, read, write, append, checking existence, and delete. The with statement is the preferred method for file handling.

Example 1: File Write and Read
file_name = "data.txt" # Writing using the 'with' statement (ensures file is closed) with open(file_name, 'w') as f: f.write("Hello File I/O!\n") f.write("Second Line.") # Reading using 'with' statement with open(file_name, 'r') as f: content = f.read() print(content) # Output: # Hello File I/O! # Second Line.

File Modes:

  • ‚Ä¢ 'r' - Read (default)
  • ‚Ä¢ 'w' - Write (creates new file or overwrites existing)
  • ‚Ä¢ 'a' - Append (adds to end of file)
  • ‚Ä¢ 'x' - Create (fails if file exists)

B. Console Input and Output (Objective 3.2)

Reading input from the console and printing formatted text are essential skills.

Example 2: Formatted Output
name = "Python" version = 3 # string.format() method message1 = "Language: {}, Version: {}.".format(name, version) print(message1) # f-String method (recommended - cleaner and faster) message2 = f"Language: {name}, Version: {version}." print(message2) # Both Output: Language: Python, Version: 3.

C. Code Documentation and Structure (Objectives 4.1 & 4.2)

Good documentation uses indentation, white space, comments, and docstrings. Functions are defined using def and can use return.

Example 3: Function Definition and Documentation
# Use comments for short explanations def calculate_area(length, width=1): """ Calculates the area of a rectangle. (This is a Docstring) :param length: The length of the rectangle. :param width: The width of the rectangle (defaults to 1). :return: The calculated area. """ area = length * width return area # Function Calls area1 = calculate_area(10) area2 = calculate_area(5, 5) print(area1) # 10 (uses default width=1) print(area2) # 25 (5 * 5)
üí° Best Practices:
‚Ä¢ Use meaningful variable and function names
‚Ä¢ Add comments for complex logic
‚Ä¢ Include docstrings for all functions
‚Ä¢ Follow consistent indentation (4 spaces)
‚Ä¢ Use white space to improve readability

‚ùì Quiz: Review Check

Question 1: What file mode must be used to add content to the end of an existing file?

  • a) 'w' (write)
  • b) 'r' (read)
  • c) 'a' (append) ‚úì
  • d) 'x' (create)
Answer: c) 'a' (append)
The 'a' mode opens a file for appending. Content is added to the end without overwriting existing data.

Question 2: Which Python feature is the recommended modern way to format string output?

  • a) string concatenation (+)
  • b) f-Strings ‚úì
  • c) print()
  • d) input()
Answer: b) f-Strings
f-Strings (formatted string literals) are the modern, recommended way: f"Hello {name}"

Question 3: The primary benefit of using the with statement for file operations is:

  • a) It makes the code run faster
  • b) It automatically handles the open and close operations ‚úì
  • c) It deletes the file after reading
  • d) It checks for file existence
Answer: b) It automatically handles the open and close operations
The with statement ensures files are properly closed even if errors occur, preventing resource leaks.

Question 4: A function parameter specified as def example(x, y=10) has a:

  • a) Syntax error
  • b) Required argument
  • c) Default value ‚úì
  • d) return value
Answer: c) Default value
The parameter y has a default value of 10. If no value is provided when calling the function, 10 will be used.

Question 5: What tool can be used to generate documentation from docstrings?

  • a) pip
  • b) pydoc ‚úì
  • c) def
  • d) unittest
Answer: b) pydoc
pydoc is Python's documentation generator tool that extracts docstrings to create formatted documentation.
