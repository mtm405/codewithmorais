üí° Key Vocabulary

Term Definition Context/Example
Syntax Error An error that occurs when the code violates the grammatical rules of the language Missing a colon or parentheses
Logic Error An error where the code is syntactically correct but produces an unintended or incorrect result Using multiplication when division was intended
Runtime Error (Exception) An error that occurs while the program is running, causing it to crash TypeError, ZeroDivisionError, FileNotFoundError
Exception Handling The process of responding to runtime errors in a controlled way to prevent the program from crashing try, except, else, finally
raise A keyword used to manually trigger an exception if age < 0: raise ValueError("Age cannot be negative")
Unit Testing A software testing method where individual units (functions, methods) are tested to ensure they are fit for use Using the unittest module

üìù Detailed Information and Examples

A. Error Analysis and Fixing (Objective 5.1)

You must be able to differentiate between the three main types of errors and fix them:

Example 1: Error Identification
Error Type Code Example Fix
Syntax Error if x > 10 Add a colon: if x > 10:
Logic Error avg = sum(data) * len(data) Use division: avg = sum(data) / len(data)
Runtime Error print(10 / 0) (ZeroDivisionError) Use try/except to handle it

B. Exception Handling (Objective 5.2)

The try/except/else/finally block allows for graceful error management.

Example 2: Exception Handling with try/except/finally
try: num1 = int(input("Enter number 1: ")) num2 = int(input("Enter number 2: ")) result = num1 / num2 except ValueError: print("Invalid input. Please enter integers only.") except ZeroDivisionError: print("Cannot divide by zero!") else: # Runs only if NO exception occurred print(f"Result is: {result}") finally: # Always runs for cleanup print("Execution attempt finished.")
üí° Exception Handling Flow:
‚Ä¢ try: Code that might cause an error
‚Ä¢ except: Handle specific error types
‚Ä¢ else: Runs if no errors occurred
‚Ä¢ finally: Always runs (cleanup code)

C. Unit Testing (Objective 5.3)

Using the unittest module is standard for unit testing, employing assert methods to check expected outcomes.

Example 3: Unit Testing (Conceptual)
import unittest # Function to test def add_numbers(a, b): return a + b # Test class inherits from unittest.TestCase class TestMath(unittest.TestCase): def test_addition(self): result = add_numbers(2, 3) self.assertEqual(result, 5) def test_type(self): result = add_numbers(1, 1) self.assertIsInstance(result, int)

Common assert methods:

  • ‚Ä¢ assertEqual(a, b) - Check if a == b
  • ‚Ä¢ assertTrue(x) - Check if x is True
  • ‚Ä¢ assertIsInstance(a, type) - Check if a is of type
  • ‚Ä¢ assertRaises(exception) - Check if exception is raised

‚ùì Quiz: Review Check

Question 1: What type of error is typically raised when you try to access a key that does not exist in a dictionary?

  • a) Syntax Error
  • b) Logic Error
  • c) Runtime Error (Exception) ‚úì
  • d) Indentation Error
Answer: c) Runtime Error (Exception)
Accessing a non-existent dictionary key raises a KeyError, which is a runtime exception.

Question 2: Which keyword must you use to manually terminate a program's execution by generating an exception?

  • a) throw
  • b) except
  • c) finally
  • d) raise ‚úì
Answer: d) raise
The raise keyword is used to manually trigger an exception.

Question 3: The assertEqual method is part of which Python testing framework?

  • a) pydoc
  • b) os
  • c) unittest ‚úì
  • d) sys
Answer: c) unittest
assertEqual is an assertion method from Python's unittest module.

Question 4: Code in the else block of a try...except structure is executed when:

  • a) An exception occurs
  • b) No exception occurs ‚úì
  • c) The program terminates
  • d) The finally block finishes
Answer: b) No exception occurs
The else block runs only if the try block completes without raising an exception.

Question 5: What is the purpose of a Logic Error?

  • a) To crash the program
  • b) To violate Python's syntax rules
  • c) To produce an incorrect result despite correct syntax ‚úì
  • d) To prevent the program from starting
Answer: c) To produce an incorrect result despite correct syntax
Logic errors don't cause crashes or syntax errors, but they produce wrong results because the logic is flawed.
