Slide 2

Flow Control: The Success Path ‚úÖ

üìå Scenario: The code in the try block works perfectly.
Keyword Function Runs?
try Tests the code. YES
except Skips the error handler. NO
else Code runs ONLY if try succeeds. YES
finally Code runs NO MATTER WHAT. YES

Code Example: Success Flow

y = 10
try:
  print("Try: Success!")
except:
  print("Except: An error occurred.")
else:
  print("Else: Success code runs.")
finally:
  print("Finally: Cleanup always runs.")

üì§ Output:

Try: Success!
Else: Success code runs.
Finally: Cleanup always runs.
Slide 3

Flow Control: The Failure Path üõë

üìå Scenario: The code in the try block fails (an exception is raised).
Keyword Function Runs?
try Fails and raises the error. PARTIALLY
except Catches the error and handles it. YES
else Skips the success code. NO
finally Code runs NO MATTER WHAT. YES

Code Example: Failure Flow

try:
  print("Try: Trying to print undefined 'z'")
  print(z)  # <-- Fails here!
except:
  print("Except: Error was caught.")
else:
  print("Else: Success code runs.")
finally:
  print("Finally: Cleanup always runs.")

üì§ Output:

Try: Trying to print undefined 'z'
Except: Error was caught.
Finally: Cleanup always runs.

Specific Exception Types üßê

üéØ Goal: Don't just catch any error; catch the right error.
Exception Type When It Happens Example Code
NameError Variable or function name is not defined. print(my_pet)
TypeError Operation uses the wrong data types. 5 + "text"
ValueError Correct type, but an invalid value. int("hello")
ZeroDivisionError Trying to divide any number by zero. 10 / 0

Code Example: Multiple Exceptions

a = 10
b = 0
c = "text"

try:
  # Rerun: What if b=1? (It causes the TypeError next)
  result = a / b
  result_sum = a + c

except ZeroDivisionError:
  print("üõë Error: Cannot divide by zero.")

except TypeError:
  print("üî† Error: Cannot mix numbers and text.")

üì§ Output (with b=0):

üõë Error: Cannot divide by zero.

üì§ Output (if b=1):

üî† Error: Cannot mix numbers and text.
Slide 5

Raising Exceptions üö®

üéØ Goal: Learn how to intentionally throw (raise) exceptions in your code.

What is raise?

As a Python developer, you can choose to throw an exception if a condition occurs.
To throw (or raise) an exception, use the raise keyword.
You can define what kind of error to raise and the text to print to the user.

Example 1: Raise a Generic Exception

x = -1

if x < 0:
  raise Exception("Sorry, no numbers below zero")

üì§ Output:

Exception: Sorry, no numbers below zero

Example 2: Raise a Specific Exception Type

x = "hello"

if not type(x) is int:
  raise TypeError("Only integers are allowed")

üì§ Output:

TypeError: Only integers are allowed
üí° When to use raise: Input validation, enforcing business rules, custom error messages for users
