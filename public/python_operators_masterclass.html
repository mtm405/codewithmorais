üßêUnderstanding The Remainder (%)

The modulus operator returns what is left over after you divide an integer evenly.

Calculate Remainder: What is 14 % 4?

  • Step 1: Divide normally: 14 √∑ 4 = 3.5
  • Step 2: Take the whole number: 3
  • Step 3: Multiply back: 3 √ó 4 = 12
  • Step 4: Subtract: 14 - 12 = 2

Result: The remainder is 2

print(14 % 4) # Output: 2 print(17 % 5) # Output: 2 print(20 % 6) # Output: 2

‚ûóThe Two Division Operators: / vs //

Don't confuse these! They work very differently.

/ Regular Division

Returns a DECIMAL (float)

print(10 / 3) # 3.3333... print(20 / 4) # 5.0 print(7 / 2) # 3.5

// Floor Division

Returns a WHOLE NUMBER (int)

print(10 // 3) # 3 print(20 // 4) # 5 print(7 // 2) # 3

üîë Key Difference:

10 / 3 = 3.333... (keeps the decimal)
10 // 3 = 3 (drops everything after the decimal, rounds DOWN)

üí° Use Cases:

Use / when you need exact decimals (prices, measurements)
Use // when you need whole numbers (converting time, splitting items equally)

Identity Operators

Definition: Compare memory location of objects (are they the same object in memory?)

is
Same Object
is not
Different Objects

Key: Identity ‚â† Equality

a = [1, 2, 3] b = a # Reference c = [1, 2, 3] # New object print(a == b) # True (same value) print(a is b) # True (same object) print(a == c) # True (same value) print(a is c) # False (different objects)

üîçUnderstanding Your Code

What does the code do?

It checks if two variables point to the exact same object in computer memory.

Which operators did we use?

  • is (Identity): original is copy ‚Üí True because copy points to the same list object as original

Important difference:

  • == (Equals): Checks if VALUES are the same
  • is (Identity): Checks if they're the SAME OBJECT in memory

üí° Key Insight: Two lists can have the same values but be different objects!

Bitwise Operators

Definition: Operate on numbers at the binary level (bits).

&
AND
|
OR
^
XOR
~
NOT
<<
Left Shift
>>
Right Shift

Note: Bitwise operators work with binary representations. Useful for flags and low-level operations.

üîçUnderstanding Your Code

What does the code do?

It performs a bitwise OR operation on numbers 5 and 3.

Understanding the operation:

  • 5 in binary: 101
  • 3 in binary: 011
  • | (OR operation): 5 | 3 ‚Üí 111 (which equals 7 in decimal)

Other bitwise operators:

  • &: AND - Both bits must be 1
  • ^: XOR - Bits must be different
  • ~: NOT - Flips all bits
  • <<, >>: Shift bits left or right

üí° Key Insight: Bitwise operators work at the binary level, useful for advanced programming like game development!
