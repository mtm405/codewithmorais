üîî Bell Ringer: The Dangers of File I/O (5 Minutes)

Question:

A critical bug report states that every time a user saves a new high score, all previous high scores are erased. Analyze the code snippet below and identify the mistake and the correct solution.

# File currently contains: "High Score: 500\n"

# Programmer runs this code:
new_score = 650
with open('scores.txt', 'w') as f:
    f.write(f'New High Score: {new_score}\n')

# What is the mistake and the correct mode to prevent data loss?
Answer & Takeaway

The Mistake: The programmer used the 'w' (write) mode.

The Consequence: The 'w' mode's primary function is to create a new file. If the file already exists, it is immediately truncated (emptied), leading to total data loss.

The Solution: The programmer should have used the 'a' (append) mode to add the new high score to the end of the existing file.

üìù Section 1: Structure, Modes, and Reading Methods

1.1 The Golden Rule: The Context Manager (with open())

The most fundamental rule of File I/O is using the with open() structure.

Element Purpose Why It's Mandatory (Q22)
with Initiates the context manager. Guarantees the file operation is handled safely.
open(...) Opens the file, specifying its path and mode. Required to establish the file connection.
as f: Assigns the active file object to the variable f. Provides a reference to work with the file.
Implicit Close Occurs automatically when the indented block ends. Prevents resource leaks and data corruption by ensuring f.close() is called, even if an error occurs.

1.2 File Modes: The W-R-A System

The mode argument is arguably the most dangerous aspect of File I/O. Choose the wrong one, and you risk losing data or crashing your program.

Mode Name Behavior Crucial Note
'w' Write Creates a file. If it exists, it is COMPLETELY ERASED (overwritten). Use ONLY when replacement is intended.
'a' Append Adds new data to the very end of the existing file. You MUST manually add the newline character (\n) for neat records.
'r' Read Opens file for reading only. RISK: Crashes with a FileNotFoundError if the path is wrong or the file is missing.

1.3 Essential Reading Methods

Once a file is open in 'r' mode, the file pointer starts at the beginning. You choose how to move it to extract data.

Method What it Returns How it Affects the File Pointer Use Case
f.read() The entire file content as a single string. Moves the pointer to the end of the file. Configuration files, small documents.
f.readline() Only the next line as a string (Q23). Moves the pointer to the start of the next line. Processing large logs one line at a time.
f.readlines() A list of strings (one element per line). Moves the pointer to the end of the file. Iterating over all lines quickly.

Example:

# 1. Writing (Overwrites the file)
with open('log.txt', 'w') as f:
    f.write('--- SESSION LOG ---\n')
    f.write('App started successfully.\n')

# 2. Appending (Adds to the end)
with open('log.txt', 'a') as f:
    # Always include '\n' in append mode
    f.write('User logged in: Alice\n') 

# 3. Reading (Demonstrates readline vs. read)
with open('log.txt', 'r') as f:
    # Reading the first line (Header)
    header = f.readline().strip() 
    print(f"Header Found: {header}")
    
    # Reading the rest of the file from the current pointer position
    remaining_log = f.read()
    print(f"\nRemaining Log Entries:\n{remaining_log}")

üìù Section 2: File Safety and Formatted Output

2.1 The Safety Routine: Preventing Crashes (Q23)

Since 'r' mode is fragile, we must use os.path.exists() to check if the file is present before attempting to read. This is crucial for graceful error handling.

The Logic: If the file is missing, we use 'w' mode to create it with a header. If it exists, we use 'a' mode.

import os 
import time

def safe_database_update(filename, data_record):
    # STEP 1: Check if the file exists to determine the correct mode
    if os.path.exists(filename):
        # PATH A: File exists - APPEND mode ('a') is safe for adding data.
        mode = 'a'
        entry = f"{data_record}\n"
        print(f"Status: Appending to existing file '{filename}'")
    else:
        # PATH B: File missing - WRITE mode ('w') is used to CREATE the file.
        mode = 'w'
        header = "--- RECORD ID | DATA | TIMESTAMP ---\n"
        entry = f"{header}{data_record}\n"
        print(f"Status: Creating new file '{filename}'")

    # STEP 2: Execute the I/O operation inside the Context Manager
    try:
        with open(filename, mode) as f:
            f.write(entry)
    except Exception as e:
        print(f"FATAL ERROR during file write: {e}")

# Example Usage (Run this script twice to see both paths executed):
record_id = 1001
data = "System check OK"
timestamp = time.strftime('%H:%M:%S')

safe_database_update('system_data.db', f"{record_id} | {data} | {timestamp}")

2.2 Advanced Output Formatting for Files

When writing numerical data to a file (like an invoice), you must use F-string specifiers to maintain precision and professional alignment.

Specifier Description Example Result
:.2f Fixed-point float, 2 decimal places. (Q39) f"{price:.2f}" 12.56 (12.559 rounds up)
:>15 Right-alignment in a field of 15 characters. f"{total:>15.2f}"           12.56

Example:

import os
import time

filename = 'transactions.csv'

# --- 1. Safety Check and Header Creation ---
if not os.path.exists(filename):
    with open(filename, 'w') as f:
        f.write("Transaction,Amount,Time\n")
    print(f"CREATED: New file '{filename}' initialized with header.")


# --- 2. Data Capture and Formatting (Requires Casting) ---
try:
    transaction_name = input("Enter transaction description (e.g., Sale, Refund): ")
    # CRITICAL: Must cast input to float for precise numerical handling
    amount = float(input("Enter transaction amount (e.g., 50.75): ")) 
    
    timestamp = time.strftime('%H:%M:%S')

    # Format the amount to two decimal places for financial reporting
    formatted_amount = f"{amount:.2f}" 

    # Prepare the line to be written (CSV format, ready to be appended)
    new_record = f"{transaction_name},{formatted_amount},{timestamp}\n"

    # --- 3. Append to File ---
    with open(filename, 'a') as f:
        f.write(new_record)
        
    print(f"\nSUCCESS: Added record to {filename}: {new_record.strip()}")

except ValueError:
    print("\nERROR: Invalid input. The amount must be a number.")
except Exception as e:
    print(f"\nAn unexpected error occurred: {e}")
Key Takeaway: Always use os.path.exists() to check for file existence before reading, and always use proper formatting specifiers when writing numerical data to ensure precision and readability.
