1. What is a Module?

Think of a Module like a DLC pack for a video game or a Toolbox for a mechanic.

  • Python (Standard): Comes with the basics (loops, variables, functions).
  • Module: A separate file filled with extra tools, functions, and code that someone else (or you) wrote.

The Goal: You don't want to write code from scratch to calculate square roots or shuffle a deck of cards. Just "plug in" a module that already does it!

Analogy: If main.py is your smartphone, a Module is an App you install to do specific cool stuff.

2. Why do we need them?

  • Laziness is good (DRY Principle): "Don't Repeat Yourself." If you write a cool function, save it in a module so you can use it in 100 different programs without typing it again.
  • Organization: Imagine if all the songs on Spotify were in one giant list. Nightmare, right? Modules act like Playlists to keep related code together.

3. The "Big 3" Import Commands

To use a module, you have to "connect" it to your program using the import keyword.

A. The Standard Import (The Safe Bet)

Imports the whole toolbox. You have to use the toolbox name to get a tool.

import math print(math.sqrt(25)) # You must type 'math.' first!

B. The Alias Import (The Nickname)

Imports the toolbox but gives it a shorter nickname so you type less.

import random as r num = r.randint(1, 10) # fast and clean

C. The Specific Import (The Sniper)

Grabs ONLY the specific tool you need. You don't need to type the module name anymore.

from time import sleep sleep(2) # No need to type 'time.sleep'

4. Built-in Modules You Should Know

Python comes with these installed ("Batteries Included"). You don't need to download anything.

Module What it does Cool Command
random RNG (Random Number Generator) for games. random.choice(['heads', 'tails'])
time Pauses code or tracks real-world time. time.sleep(5) (pauses for 5 sec)
math Advanced math (Pi, square roots, trig). math.pi
os Controls your computer (files, folders). os.system("clear") (clears screen)
turtle Draws graphics on screen. turtle.forward(100)

5. How to Write Your Own Module

It's easier than you think. It's just a Python file!

Step 1: Create the Module Make a file named my_tools.py. Put a function in it.
# File: my_tools.py def say_hello(name): return f"Yo, {name}!"
Step 2: Use It Make a file named main.py in the SAME FOLDER.
# File: main.py import my_tools print(my_tools.say_hello("Class"))

6. Pro-Tip: The __pycache__ Folder

Question: "I ran my code and this weird folder appeared. Did I break it?"

Answer: NO! That is Python's way of "optimizing" your module so it loads faster next time. Ignore it. Do not delete it (it will just come back).
